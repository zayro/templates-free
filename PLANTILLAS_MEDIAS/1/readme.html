About The Theme - top

This theme is designed to be easy to use for either static HTML pages, or to convert for a content management system (CMS). The design is broken down to allow each section to be separated into a self-contained content block. These blocks are marked with comment tags to make finding them easier: 

<!-- Name of Section or Area -->
<div id="TheIdOfTheContainer" >
	Your HTML Content.
</div>

Layouts - sample-layout.html

There are examples of the different layouts and content configurations in the sample pages. Detailed information can be found on the "Layout" page (sample-layout.html) included in the sample files.

Home Page - You must include the class "home" in the <body> of your home page:

<body class="home">

Alternate Layout - You must include the class "alternate" in the <body> of your pages to use the alternate layout style:

<body class="alternate">

Or in the case of the home page:

<body class="home alternate">

Text - sample-text.html

Text formatting details are also available in the sample files. For this information view the "Text Styles" page (sample-text.html) included in the sample files. This contains example code and sample outputs of all the common HTML tags used for content editing.

Cufon - font replacement

This theme takes advantage of cufon font replacement for headings and other title elements. The font used, "Perspective Sans", is a free font available online. If you have missing characters in your titles, such as accent marks or symbols, you will need to recreate the cufon file. To do this, download the font file below, and using the cufon website, create a new font replacement JavaScript file. Include the glyphs you require for your text and replace the existing file in the theme "www/js" folder. This should not be necessary in most cases. Exceptions may include the need for Russian characters, etc.

Download "Perspective Sans"

Layout and Design Structure - top

This theme is a fixed-width design with pre-configured styles for creating layouts with one, two or three columns.

The default content area is commented to make it easier to find. Look for the HTML comment with the text "Page Content". These comments will help you identify the function of the different areas of the code.

<!-- Page Content -->

If you would like to edit the color, font, or style of any elements in one of these columns, you would do the following:

.contentArea a {
	color: [color];
}

If you find that your new style is not overriding, it is most likely because of a specificity problem. Scroll down in your CSS file and make sure that there isn't a similar style that has more weight.

I.E.

body .contentArea a {
	color: [color];
}

To ensure that your new styles are applied, make sure that they carry enough "weight" and that there isn't a style lower in the CSS file that is being applied after yours.

The HTML code is indented to make it easier to edit. There are also comment markers as a guides to find the beginning and end of major sections such as the top, header, slide show, page content and footer. Each of these sections is labeled with the appropriate tags and has a corresponding ID tag of the main container for that section.

You should use the included sample files as templates for your own layouts. Be sure to backup a file before editing it, or create a copy and work from it.

CSS Files and Structure - top

There are several CSS files included with this theme, but only a few will be necessary for customization. The CSS files are all located in the "/www/css" folder. Within that folder there are two sub-folders, "source" and "skins".

The first folder, "source", contains the original CSS files with formatted code and comments. Only a few files are in this folder. Some of these are the original CSS files for add on scripts used in the design. If you find a CSS file that is uncommented and you need the original, check this folder.

The second folder, "skins", houses the skin specific style sheet for each custom skin. If you create your own custom skin, it is recommended you save the style sheet in this folder.

Primary CSS Files

default.css (required)

This file provides the foundation for the site design and all skins. It controls the layout and design structure of the theme. Making modifications to this file will usually effect all skins unless it is a color or image related change. The skin files act as an extension of "default.css" to apply custom colors and graphics. Before you edit this file you should consider making a custom skin style sheet or editing the one you are already using for your design.

/skins/skin-#.css

These are the skin specific style sheets. For the purposes of the online demo, these files are named numerically in a format such as, "skin-#.css". If you plan to continue using the dynamic style switching functions that are included in the demo, you must name your skin files this way. If you are going to use a single skin you can name the file however you like, and even save it in a different folder if you choose.

Skinning the Theme - top

Each color style, "skin", is controlled by a custom CSS file. These are located in "/www/css/skins". If you want to keep the ability to use different colors and change them on the fly you will want to maintain this structure. The images for each skin are saved in a folder under "/www/images/skins/skin-123/" where "skin-123" is the name of your CSS file. To create a custom skin, duplicate an existing skin CSS files and edit it as desired. You can easily output the custom graphics for you skin using one of the "Theme Graphics - for skinning.PSD" files located in the "Design\Files for skinning" folded included with the theme.

After creating the skin CSS file and a matching images folder, save your images to this folder. Use the "Save for web..." feature of Photoshop and select "All user slices" to get the desired graphics. You can create a custom skin in a few minutes by modifying the background colors of the main layers in this file. If you want to get more detailed and tweak the output you can edit the individual highlights and shadow features of the layers above the backgrounds. Play with the different layers to see what effect they have. The file was designed to allow easy and fast skinning.

After you have the new skin CSS file and images folder created, update the reference in your HTML file's <HEAD> tag of each HTML page to use this new skin (see below for details).

Dynamic Skin Changer

If you plan to use the included skin switcher you don't need to change the CSS link in the <HEAD> section of your HTML files. You can add a thumbnail graphic to the slider and it will show your new skin in the animation. If you have increased the total number of skins and intend to keep the skin slider, open "demo.js" and edit the total number of skins line: var skinCnt = 5;

IMPORTANT: If you are going to create a custom skin or do any advanced code editing, you will likely need to edit the JavaScript loading file, "js/onLoad.js" This file controls loading and executing of many dynamic JavaScript functions used in the theme.

To create a single skin for your site

The best way to go about creating your own skin, is by starting from one of the pre-made styles that come with the theme. An easy way to do this would be as follows:

  1. First, create a copy of an existing style sheet from "/www/css/skins/skin-#.css", and increment the # to the next unused value.
  2. Create a duplicate of the same skin's images folder located in "/www/images/skins/skin-#"
  3. Give your new images folder the same name as the CSS file by incrementing the number to match.
  4. Open the "Theme Graphics - for skinning.psd" file in the "Design/Files for skinning" folder and modify the colors to create your desired graphics.
  5. Output the graphics using "Save for web..." to the new skin's images folder. Choose "All user created slices" as the option for output method when asked.
  6. Open the new CSS file for your skin. Do a find and replace for the path to your images folder, this would be something like, Find: "skin-5", Replace with: "skin-6", to update all occurrences of the image folder path in the file. Save when finished.
  7. Update the link in your HTML files pointing to your new CSS skin file:
    <!-- Skin Style Sheet -->
<link rel="stylesheet" href="css/skins/skin-1.css" type="text/css" id="SkinCSS" />
    (change the file name from "skin-1.css" to the name of your new skin file)
  8. Save your HTML files and load the page to preview the new skin.
  9. Open your skin CSS file and begin editing the individual style properties to match the graphics you have created for your new skin.
  10. Save, upload and enjoy!

Font Shadows and Highlights

After you have created your skin, or configured the theme to work with the skin of your choice, you may notice that font highlights and shadows no longer match your colors. These are a feature of the cufon font replacement. To edit these, open "js/onLoad.js" and scroll to the very end of the file. These styles can be configured from here. If you want to use an existing skin and need the styles from the demo site, open "js/demo.js" and scroll to the very end of the file. You will find a customized version of these styles here. One exists for each included skin. Paste the desired styles in to "onLoad.js", replacing the existing calls to cufon and your are ready to go.

Contact Form - top

Contact form setup

The theme includes a working contact form. This form uses PHP and requires a server running PHP for it to work.

Adding Your Email Address

To add your own email address to the form, open the file "contact-send.php" and at the very top of the file replace "yourname@yourwebsite.com" with your own email address. Be sure not to delete the quotes, semi-colon or other code when making this change.

$YourEmailAddress = "yourname@yourwebsite.com";

Problems Sending Emails

If you experience trouble sending emails you should verify the following:

  1. Make sure the address entered in "contact-send.php" is correct.
  2. Verify that PHP is installed on the server your site is running from.
  3. If necessary contact your hosting provider to ensure they do not need to activate something.

Pages with Contact Forms

You must include the following code in the <HEAD> tag of any page with the contact form.

<!-- form validation scripts (for contact page) -->
<script src="js/jquery.validate.min.js" type="text/javascript"></script>
	
<!-- Contact form triggers and AJAX submit -->
<script src="js/contactForm.js" type="text/javascript"></script>

For and example of this review the included contact form sample page.

Special Features - top

Alternate Layouts

The theme includes 2 layout styles. The default style will be what displays normally. To use the alternate layout style which includes a header background color, add the class "alternate" to the <body> tag of your pages.

<body class="alternate">

Button Style Disabling

The custom button styles will automatically apply to any <button> or <input type="[submit or button]"> on your page. In some situations you may need a button not to have these styles to preserve special functionality. You can add the class "noStyle" to any button and it will be ignored by the button styling function.

<button class="noStyle">Do Not Style This Button</button>

Specialty Links

You can add several special types of links to tap into built in functionality of the theme.

Image icons for menu links

To add an icon to a menu item, you must do two things. First you need to have the image defined in your style sheet. For an example of this open the default style sheet and search for ".mmFeeds". The code below shows the basic formatting of these styles.

/* icons for menu items */

.mmLogin	{ background: url(...bullet-lock.png)...
.mmFeeds	{ background: url(...bullet-feed.png)...
.mmRSS		{ background: url(...icons/feed.png)...
.mmFacebook	{ background: url(...icons/facebook.png)...
.mmTwitter	{ background: url(.../icons/twitter2.png)...

Create your own entry by copying an existing one and giving it a custom class name. After you have this entry, add the new image you entered the path for to the location you specified. Next add a <span> tag around your menu item title resembling the following:

<span class="mmFeeds">Feeds</span>

Where the "Feeds" text is your menu item text and "mmFeeds" class is the class of your custom menu icon class created in the style sheet.

Login pages, top slide and pop-ups

You can add links that activate the custom login pop-up or the slide open top area. Include the path to the page you want to open and append one of the following to choose the function.

Pop-up Login (#login): To create a link that opens the login pop-up, at the end of your login file URL add "#login". Alternately, you can also add the class "login" to any link to achieve the same functionality. See the sample files main menu section for an example of this method.

Slide Open Top (#topReveal): To create a link that opens the slide top, enter "#topReveal" in your link's URL, or add the class "topReveal" to any link. Review the main menu included in the sample files for an example of this method.

Any content as pop-up (class="iframe"): You can optionally enter any URL and add the class "iframe" and it will open in a pop-up. This can be an internal or external path, for example:

<a href="http://www.google.com" class="iframe">Google</a>

This can be useful for linking to maps or other informational content.
